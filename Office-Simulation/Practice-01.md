# 🏢 Office Simulation Practice 01 – Backup & Compression

## Scenario

Acted as a Junior Linux System Administrator and completed simulated office support tickets related to backups, archiving, and file compression.

## Tasks Completed

### Ticket 001 – Create Backup
- Created a logs directory.
- Created multiple log files (auth.log, nginx.log, systems.log).
- Added sample log data.
- Archived the directory using tar.
- Compressed the archive using bzip2.
- Verified the backup file.

### Ticket 002 – Recover Backup
- Decompressed the .tar.bz2 archive using bunzip2.
- Listed archive contents using tar -tvf.
- Extracted files using tar -xvf.
- Verified the recovered files.

### Ticket 003 – Client Request
- Converted the backup from .tar.bz2 to .tar.gz.
- Used gzip for faster compression.
- Verified the new archive.

## Skills Practiced

- File archiving
- File compression
- Backup verification
- Data recovery
- Archive extraction
- Basic Linux troubleshooting
- Using aliases for command efficiency

## Lessons Learned

- tar creates archives but does not significantly compress files.
- gzip is faster, while bzip2 provides better compression.
- Never use cat on a .tar archive because it is a binary file.
- Use tar -tvf to view archive contents.
- Use tar -xvf to extract files from an archive.
